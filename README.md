# SHU Agentic RAG Assistant

A conversational AI assistant for Sheffield Hallam University (SHU), built with a layered agentic RAG architecture. The assistant answers student questions about courses, modules, fees, accommodation, admissions, and campus life using an official SHU knowledge base — and falls back to Tavily web search for anything not covered.

---

## Architecture

```
User query
    │
    ▼
SHUAgent  (agent/agent.py)
    │
    ├── AgentCore Short-Term Memory   ← conversation history across turns
    │
    ├── Tool: shu_knowledge_base      ← Chroma vectorstore (SHU scraped data)
    │       │
    │       ├── Query Processor       ← Amazon Nova Lite (structured intent + filters)
    │       ├── Retriever             ← Chroma similarity / MMR search + fallback
    │       └── Reranker              ← CrossEncoder ms-marco-MiniLM-L-6-v2
    │
    └── Tool: web_search              ← Tavily (fallback for missing KB data)
    │
    ▼
Answer  (Amazon Nova Lite generator)
```

## Configuration

All settings live in `config/settings.py`. Key values:

| Setting | Default | Description |
|---|---|---|
| `EMBEDDING_MODEL` | `all-MiniLM-L6-v2` | Sentence-transformer embedding model |
| `EMBEDDING_DEVICE` | `cpu` | Change to `cuda` if GPU is available |
| `QUERY_MODEL_ID` | `amazon.nova-lite-v1:0` | Bedrock model for query classification |
| `GENERATOR_MODEL_ID` | `amazon.nova-lite-v1:0` | Bedrock model for answer generation |
| `GENERATOR_TEMPERATURE` | `0.3` | Generator LLM temperature |
| `RERANKER_MODEL` | `ms-marco-MiniLM-L-6-v2` | CrossEncoder reranker model |
| `DEFAULT_K` | `12` | Default number of docs to retrieve |
| `MAX_K` | `40` | Max docs for broad list queries |
| `AGENT_MAX_ITERATIONS` | `5` | Safety limit on agent reasoning steps |
| `LANGCHAIN_PROJECT` | `shu-rag-assistant` | LangSmith project name |

## How Retrieval Works

The RAG tool uses a three-level fallback strategy to ensure it always returns results:

1. **Primary** — full metadata filter (chunk type + course + study mode etc.)
2. **Fallback** — chunk type filter only (drops user-specified fields if primary returns nothing)
3. **Last resort** — pure vector similarity search with no filter

The query processor (Amazon Nova Lite) classifies each query into one of three intent types before retrieval:

- `course_summary` — fees, UCAS codes, entry requirements, placement, campus
- `module_detail` — module names, credits, assessment, compulsory vs optional
- `general` — accommodation, admissions, contacts, student life, support

Retrieved documents are then reranked by a CrossEncoder before being passed to the generator.

## LangSmith Tracing

Tracing is enabled by default when `LANGCHAIN_API_KEY` is set. Every agent turn is automatically traced with child spans for tool selection, tool execution, and answer generation — no manual decorators needed.

To disable tracing, set in `config/settings.py`:

```python
LANGCHAIN_TRACING_V2 = False
```

## AgentCore Short-Term Memory

The agent uses AWS AgentCore to persist conversation history across turns within a session. This replaces the in-memory chat history list and allows the agent to maintain context even if the process restarts mid-session.

### Setup

```bash
# 1. Create a memory resource
aws bedrock-agentcore create-memory --name shu-rag-memory --region us-east-1

# 2. Copy the returned memoryId into config/settings.py
AGENTCORE_MEMORY_ID = "mem-xxxxxxxxxxxxxxxxx"
```

IAM permissions required on your Bedrock role:

```json
bedrock-agentcore:CreateEvent
bedrock-agentcore:GetMemory
bedrock-agentcore:RetrieveMemoryRecords
```

If `AGENTCORE_MEMORY_ID` is left empty, the agent falls back to a simple in-memory list automatically — no code changes needed.

## Evaluation (RAGAS)

Run the evaluation suite against 10 built-in test questions:

```bash
python evaluate.py

# Run a subset and save to a custom file
python evaluate.py --sample 5 --output results/run1.csv
```

Four metrics are measured:

| Metric | What it measures |
|---|---|
| `faithfulness` | Are answers grounded in retrieved context? |
| `answer_relevancy` | Does the answer address the question? |
| `context_precision` | Are the most relevant chunks ranked highest? |
| `context_recall` | Does the retrieved context cover the ground truth? |

Evaluation runs retrieval and generation directly — no agent loop — so each question is scored independently without cross-turn contamination.

## Known Limitations

- The knowledge base was scraped in March 2026. Course details, fees, and entry requirements may have changed — always verify at [shu.ac.uk](https://www.shu.ac.uk).
- Some MSc course variants (e.g. part-time, distance learning) may have incomplete data in the knowledge base. Web search will be used as fallback for these.
- The embedding model (`all-MiniLM-L6-v2`) runs on CPU by default. For faster ingestion and retrieval, set `EMBEDDING_DEVICE = "cuda"` if a GPU is available.
